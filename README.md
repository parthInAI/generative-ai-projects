## Why I Built These 4

There is something quietly astonishing about a model that generates things that did not exist before. Not copying. Not retrieving. Actually making.

These four projects are not here to show off. They are here to make sense of the underlying ideas that are powering the most significant shift in creative technology in a generation -- text-to-image systems, AI-generated video, voice synthesis, drug discovery. All of it traces back to a small set of fundamental questions: How do you teach a machine to understand the structure of data deeply enough to invent more of it? How do you make that process stable and controllable? And what do you do when the machine gets so good at inventing that you can no longer trust what you see?

Each project addresses one of those questions directly.

## The Story Behind Each Project

### 01 - Variational Autoencoder -- Teaching a Machine to Imagine

Before you can generate something new, you have to understand what makes existing things the way they are. The VAE does this by compressing data into a compact latent space -- a kind of coordinate system where nearby points produce similar-looking outputs. The key insight is forcing that space to be smooth and continuous, so that every point in it decodes to something meaningful. This is not just useful for generation. It is the foundation for anomaly detection, drug discovery, and every creative AI tool that lets you "interpolate" between two concepts. Stable Diffusion's latent space is a direct descendant of this idea.

---

### 02 - Generative Adversarial Network -- Two Networks, One Goal

The GAN is one of the most elegant ideas in machine learning. You pit two networks against each other. The generator tries to produce fake data convincing enough to fool the discriminator. The discriminator tries not to be fooled. Neither can win permanently, so they both have to keep improving -- and somewhere in that adversarial loop, the generator learns to produce things that are genuinely remarkable. This is the architecture behind the synthetic faces that went viral, the style transfer that reimagined Van Gogh and Picasso, and the data augmentation pipelines that let medical AI train on more cases than any hospital could provide.

---

### 03 - Diffusion Models -- Turning Noise into Signal, One Step at a Time

Diffusion models start from a radically simple observation: if you gradually add noise to an image until it becomes pure static, and then train a network to reverse that process one small step at a time, you get a generative model that is extraordinarily stable and expressive. There is no adversarial training, no mode collapse, no fragile equilibrium. Just a patient, iterative denoising process that can be conditioned on text, class labels, or any other signal. This is the exact mechanism inside Stable Diffusion, DALL-E 2, Midjourney, and Google's Imagen. Understanding diffusion is now a prerequisite for understanding modern generative AI.

---

### 04 - Deepfake Detection -- The Other Side of the Coin

Every technology that can create can also be used to deceive. Deepfakes -- synthetic faces and voices generated by models like the ones in this repository -- are now being used to spread misinformation, impersonate public figures, and commit fraud. This project is a response to that reality. A deepfake detector works by looking for the subtle artefacts that generative models leave behind: unnatural patterns in the frequency domain, inconsistencies in noise structure, checkerboard artefacts from transposed convolutions, over-smooth blending at face boundaries. It is a reminder that building powerful generative models responsibly means also building the tools to detect and counter them.

## Quick Start

```bash
git clone https://github.com/parthInAI/generative-ai-projects.git
cd generative-ai-projects

python -m venv .venv
source .venv/bin/activate      # Windows: .venv\Scripts\activate
pip install -r requirements.txt

mkdir -p outputs

python project_01_vae.py
python project_02_gan.py
python project_03_diffusion.py
python project_04_deepfake_detection.py
```

Each script works with or without TensorFlow. If TensorFlow is not installed,
the scripts fall back to NumPy simulations that demonstrate the same concepts
visually without training neural networks.

## Architecture Reference

```
Project 01 -- VAE

Input Image
    |
Encoder (Dense 256 -> 128)
    |           |
   mu         log_var
    |           |
    +-- z = mu + eps * exp(0.5 * log_var)  (reparameterisation)
               |
Decoder (Dense 128 -> 256 -> output)
               |
Reconstructed Image

Loss = Reconstruction Loss + KL Divergence

---

Project 02 -- GAN

Real Data -----> Discriminator <----- Generator <----- Random Noise
                     |                    |
               D tries to say         G tries to make
               "real" for real        D say "real" for fake
               "fake" for fake

---

Project 03 -- Diffusion

Forward:  x0 -> x1 -> x2 -> ... -> xT  (add noise step by step)
Reverse:  xT -> xT-1 -> ... -> x0  (U-Net predicts noise to remove)

Loss = MSE(predicted_noise, actual_noise)

---

Project 04 -- Deepfake Detection

Face Image
    |
Feature Extraction:
  -- DCT frequency energy ratio
  -- Gradient variance (texture)
  -- FFT spectrum peak analysis
  -- Colour channel correlation
  -- Noise pattern statistics
  -- Face boundary blending score
    |
Classifier (Random Forest / Gradient Boosting / Logistic Regression)
    |
Real / Fake + Confidence Score
```
